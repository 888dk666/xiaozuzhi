## 关键发现

1. **最匹配需求的项目是 MikePipe**（GitHub/bowerhaus/MikePipe，MIT 协议）：专门为"远程 Mac 听写"场景设计，通过 Tailscale 将本地麦克风音频流式传输到远程 Mac 的 BlackHole 虚拟设备，延迟 <100ms。当前仅支持 Windows 发送端，但架构可移植到 MacBook（同为 Python/PyAudio）。

2. **浏览器语音输入是最简单的"够用"方案**：Chrome/Edge 内置 Web Speech API，中文安静环境 92%+ 准确率，200-500ms 延迟。MacBook 打开网页说话 → 文字出现在网页里 → 通过剪贴板同步/共享备忘录到 Mac mini。零安装零配置，但需同 Apple ID 或手动粘贴。

3. **Android 手机转 Mac 麦克风有成熟开源方案**：AndroidMic（GPL-3.0，2026 年 2 月更新 v2.2.6）支持 WiFi/USB/ADB 三种连接方式，配合 BlackHole 虚拟声卡可在 Mac 上作为系统麦克风使用。但手机和 Mac 需同网络（WiFi 模式）或 USB 线连接。

4. **Whisper 浏览器版（WASM）可完全本地运行**：whisper.cpp 官方 WASM 版（ggerganov/whisper.cpp）、Xenova/whisper-web（HuggingFace transformers.js）均支持在 Chrome 内本地运行语音识别，无需服务器，无需网络。中文 tiny/base 模型可用，larger 模型受浏览器内存限制。

5. **Apple 官方方案有硬门槛**：macOS 内置听写（Dictation）中文准确率高且离线，但标准屏幕共享/VNC 不传音频。需要 macOS Sonoma 14 + Apple Silicon + 付费 Apple Remote Desktop（$80）开启"高性能模式"才有音频通道。两机均已满足系统要求，但需额外付费和配置。

6. **Tailscale 是跨互联网音频传输的关键基础设施**：MikePipe 已验证 Tailscale UDP 打洞可行；Helmlog 项目验证了 Tailscale 上跑 faster-whisper 的 FastAPI 服务。两个 Mac（MacBook + Mac mini）装上 Tailscale 即可组成虚拟局域网，无论物理位置。

## 方案汇总

### 方案 A：浏览器语音输入 + 文字同步（最简单，推荐小白首选）

**技术路径**：MacBook 打开 Chrome 访问语音输入网页（如 Google Docs 语音输入 / 讯飞听见网页版 / 自建 whisper-web）→ 说话 → 文字出现在网页内 → 通过 iCloud 备忘录/共享剪贴板/Google Docs 云同步到 Mac mini。

| 维度 | 评估 |
|------|------|
| 中文识别质量 | 优（Google Docs/讯飞听见均 92%+） |
| 跨设备传输方式 | iCloud 备忘录同步 / Google Docs 实时同步 / 剪贴板通用共享（同 Apple ID） |
| 延迟 | 低（语音识别 200-500ms + 云同步 1-3 秒） |
| 部署复杂度 | 低（零安装，打开浏览器即可） |
| 是否需同局域网 | 否（只需要互联网） |
| 隐私/数据流向 | 取决于使用的语音引擎：Google Docs → 音频发 Google 云端；讯飞听见 → 音频发讯飞云端；自建 whisper-web → 100% 本地浏览器内处理 |

**具体子方案**：

| 子方案 | 引擎 | 费用 | 隐私 |
|--------|------|------|------|
| Google Docs 语音输入 | Google 云端 | 免费 | 音频发送到 Google |
| 讯飞听见网页版 | 讯飞云端 | 每日 1 小时免费 | 音频发送到讯飞 |
| 自建 whisper-web（Xenova） | 本地 WASM | 免费 | 100% 本地，不传任何服务器 |
| 自建 whisper.wasm（官方） | 本地 WASM | 免费 | 100% 本地，不传任何服务器 |

**跨互联网适配**：完全适用。Google Docs 和讯飞听见本身就是云服务。自建 whisper-web 需搭配 Tailscale + 本地 HTTP 服务让 Mac mini 也能访问（或 MacBook 直接本地识别后用剪贴板同步）。

### 方案 B：MikePipe 思路 — 音频流 + 远程听写引擎（技术最优）

**技术路径**：MacBook 采集麦克风 → UDP 音频流通过 Tailscale 发送 → Mac mini 接收后注入 BlackHole 虚拟声卡 → Mac mini 内置 Dictation 或 whisper.cpp 识别 → 文字直接在 Mac mini 上。

| 维度 | 评估 |
|------|------|
| 中文识别质量 | 优（Mac 内置听写离线识别中文很准；whisper.cpp medium 模型更优） |
| 跨设备传输方式 | Tailscale UDP 音频流（16kHz, 16-bit, mono PCM, ~32KB/s） |
| 延迟 | 极低（MikePipe 实测 <100ms + 识别延迟 200-500ms = 总计 <600ms） |
| 部署复杂度 | 高（需安装 Tailscale + BlackHole + Python 脚本，当前仅 Windows 发送端，需自行适配 MacBook 发送端） |
| 是否需同局域网 | 否（Tailscale 穿透，只需互联网） |
| 隐私/数据流向 | 极优 — 音频全程经 Tailscale 加密传输，识别在 Mac mini 本地（内置听写/whisper.cpp），音频不经过任何第三方 |

**当前限制**：MikePipe 发送端仅支持 Windows。MacBook 做发送端需改写 Python 脚本（PyAudio 在 macOS 上同样可用，工作量主要是配置调试）。

### 方案 C：Android 手机当麦克风 + Mac mini 识别

**技术路径**：vivo X90 Pro+ 安装 AndroidMic App → WiFi/USB 连接到 MacBook（或直接连 Mac mini）→ 音频流到 Mac → BlackHole 虚拟声卡 → Mac mini 内置听写/whisper.cpp 识别。

| 维度 | 评估 |
|------|------|
| 中文识别质量 | 优（取决于 Mac 端识别引擎） |
| 跨设备传输方式 | AndroidMic WiFi TCP/UDP 音频流 → Mac 端接收 → BlackHole 虚拟声卡 |
| 延迟 | 低-中（WiFi 局域网 <100ms 传输 + 识别延迟） |
| 部署复杂度 | 中（安装 AndroidMic APK + Mac 客户端 + BlackHole，均为免费，但需配置 IP/端口/音频参数） |
| 是否需同局域网 | WiFi 模式需同局域网（或 USB 线直连）；跨互联网需 Tailscale + 端口转发 |
| 隐私/数据流向 | 优 — 手机→Mac 直连，音频不经过第三方。Mac 端本地识别 |

**注意**：手机作为麦克风终端时，识别引擎仍跑在 Mac mini 上。手机只做音频采集和传输。

### 方案 D：Wispr Flow（商业产品，开箱即用）

**技术路径**：MacBook 安装 Wispr Flow → 说话 → 云端识别 → 文字通过剪贴板/Flow Notes 云同步 → Mac mini（同账号登录 Wispr Flow）获取。

| 维度 | 评估 |
|------|------|
| 中文识别质量 | 优（支持 100+ 语言，含中文，AI 自动纠错、去填充词） |
| 跨设备传输方式 | Wispr Flow Notes 云同步（MacBook 说完 → 云端 → Mac mini 打开 Flow Notes 看到文字） |
| 延迟 | 低（云端识别 + 云同步，秒级） |
| 部署复杂度 | 低（下载安装 App + 注册账号，零配置） |
| 是否需同局域网 | 否（纯云服务，只需互联网） |
| 隐私/数据流向 | ⚠️ 中 — 音频发送到 Wispr 服务器（使用 OpenAI/Meta 基础设施做识别），第三方 LLM provider 30 天删除。不出售数据。音频隐私不如本地方案 |

**注意**：Flow Notes/Scratchpad 跨设备同步已在 Mac 端可用，Android 端"coming soon"。免费版 2,000 词/周，Pro $12/月。

### 补充方案：Apple Remote Desktop 高性能模式（官方方案）

**技术路径**：MacBook 通过 Apple Remote Desktop App 以"高性能模式"连接 Mac mini → 高性能模式支持音频通道 → MacBook 麦克风声音传到 Mac mini → Mac mini 内置听写直接识别。

| 维度 | 评估 |
|------|------|
| 中文识别质量 | 优（Mac 内置听写中文离线识别） |
| 跨设备传输方式 | Apple Remote Desktop 高性能模式原生音频通道 |
| 延迟 | 极低（原生协议，比 VNC 快） |
| 部署复杂度 | 低-中（购买 Apple Remote Desktop $79.99 → 两台 Mac 均需 Apple Silicon + macOS 14+ → 开启高性能模式） |
| 是否需同局域网 | 官方主要用于局域网；跨互联网需 Tailscale/端口转发（非官方支持） |
| 隐私/数据流向 | 优 — Apple 原生加密协议 |
