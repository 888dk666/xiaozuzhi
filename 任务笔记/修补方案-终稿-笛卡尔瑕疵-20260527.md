## P0 — 闭卷执行操作化

**覆盖瑕疵：** 2.1（闭卷默写偏离定义）、2.2（短期记忆非长期）、2.3（口试间隔不够）

**根因：** ops.md 步骤 6a 和步骤 7 "闭卷"的定义没有考虑会话上下文持久化的影响。Agent 摘要留在上下文里→PM 从短期记忆复述→不是检索练习。

**修补：**

1. **ops.md 步骤 6a 重写：**
   ```
   6a. 闭卷默写（retrieval practice + calibration）
       关掉所有源——包括 Agent 返回的搜索报告、对话上下文中的摘要。
       从记忆中提取，不从上下文中复述。
       默写每个核心概念的定义、关键论证、与其他概念的关系。
       每条标自信度：[确定]=能独立重构论证 / [不确定]=记得概念但无法完整重构 / [推断]=基于已有知识的合理推测未经验证。
       默写完成后才对照原文校准——自信度标了"确定"但错了→针对性重学。
       建议 6a 与 6c（写文档）之间有实质间隔（≥30 分钟）。
       无法间隔时标注"⚠️ 短期记忆污染风险"。
   ```

2. **ops.md 步骤 7 "PM 回答"段补：**
   ```
   PM 闭卷回答——不准查任何资料，包括自己刚写的学习文档。
   建议口试与文档写作之间 ≥30 分钟间隔。无法间隔时标注"⚠️ 短期记忆污染"。
   ```

3. **研究路径常见坑加：**
   ```
   - 闭卷默写走过场 — Agent 摘要还在上下文里飘着→默写测的是复述能力，不是学习效果。
     PM 自问：我是从脑子里提取的，还是从刚才读到的 Agent 返回里复述的？
   ```

## P0 — 搜索报告落盘+格式+去重

**覆盖瑕疵：** 1.1（搜索报告未落盘）、1.2（spawn prompt 未归档）、1.4（C 路时间戳异常未分析）、1.6（54 选 20 未说明筛选逻辑）、3.1（PM 学二手摘要——纯牛马须提供原文引用段）、3.8（PM 未做三路去重）

**根因：** 研究路径步骤 3 没有指定搜索报告的持久化路径和格式。Engineer agent 默认产出格式是代码导向的（"改动""自验""不确定项"），与搜索报告需求不匹配。

**修补：**

1. **ops.md 步骤 3 补产出路径：**
   ```
   每路搜索报告写入 ~/projects/晓组织/任务笔记/搜索报告-{任务名}-{路名}-{日期}.md
   ```

2. **ops.md 步骤 3 补 PM spawn 时指定搜索报告格式（关键——engineer agent 默认格式是代码导向的）：**
   PM spawn engineer 时在 prompt 中指定产出格式：
   ```
   产出格式：
   - 搜索策略：使用的关键词、信息源、筛选标准
   - 源清单：每条含 URL/来源类型(L1-L4)/全文可获取性/AI污染判据/内容摘要
   - 关键发现：3-5 条本路最重要的发现
   - 未获取项：搜索了但没拿到全文的重要源
   ```

3. **ops.md 步骤 5 显式化去重+筛选：**
   ```
   步骤5：PM 拼合搜索报告
   先执行三路去重——重叠源标注"多路覆盖"，信息冲突处标注"待交叉验证"。
   再筛选核心源——记录筛选逻辑：入选标准（L1优先/直接相关/全文可获取）、排除原因（付费墙/质量不足/重复）。
   最后整合为完整资料集。
   ```

4. **ops.md 步骤 5 补时间戳异常自检：**
   ```
   PM 拼合时自检：三路返回时间与源量是否匹配？
   某路源量显著偏多但返回偏快（如 C 路 28 源比 A 路 13 源更早完成）
   → 标注 ⚠️ 质量存疑，拼合时降权处理
   ```

5. **ops.md 步骤 3 补原文引用要求（解决 PM 学二手摘要问题）：**
   ```
   每路纯牛马对每个 L1 源标注：关键段落原文引用（至少 2 段），不只是摘要。
   PM 在步骤 6 学习转化前，至少直接阅读 3 篇 L1 核心源的关键段落原文。
   ```
   研究路径常见坑同步补：
   ```
   - 搜索报告摘要≠原文 — PM 必须读至少 3 篇 L1 一手原文的关键论证段落
   ```

6. **QA 审搜索结果（步骤 4）spawn prompt 必填字段加：** 搜索报告文件路径

7. **学习成果审查清单 #7（校准抽检）改：** 「QA 对照搜索报告文件原文验证 PM 引用准确性」（不再依赖训练数据）

## P1 — 口试机制升级

**覆盖瑕疵：** 2.4（口试单层追问——架构天花板，改用预备追问替代）、2.5（5 题全对可能是题太简单）、2.6（QA 出题一人完成）、4.5（无外部基准校准）

**根因：** 口试题设计原则缺难度梯度和外部校准。Subagent 不支持交互式追问是架构天花板。

**修补：**

1. **ops.md 口试题设计原则从 5 条扩到 7 条：**
   ```
   #6 难度梯度 — 5 题按难度递增排列：
       🟢基础题×1（核心概念定义）
       🟡中档题×3（概念间关系+论证分析）
       🔴挑战题×1（预期 PM 答不完全，用来探知识边界）
       5 题全预期 ✅→题目偏易，应调整
   #7 可外部校准 — QA 出题时每题先写好"理想答案应覆盖的要点"（不透露给 PM）。
       判定时对照要点逐条核实，保证判定标准前后一致
   ```

2. **QA 出题时（7a）加自评：**
   标注每题预期难度（🟢/🟡/🔴）。出题后自问：「5 题是否覆盖三路搜索全部维度？至少 1 道跨路综合题？难度梯度是否有效？」

3. **QA agent 定义文件口试模式补：**
   ```
   出题后自审：难度梯度是否递增？至少 1 道"挑战题"预期能探到 PM 知识边界？
   每道题先写"理想答案应覆盖的要点"（私存，不透露给 PM），判定时对照。
   ```

4. **口试单层追问替代方案（弥补 subagent 架构限制）：**
   3 次口试实验跑完后评估单层追问是否足够。如不够→QA 在 7a 出题时加"预备追问"（标注"如果 PM 在问题 X 上答得浅，补充追问 Y"），PM 在回答时一并回复。

## P1 — 自信度体系修正

**覆盖瑕疵：** 3.4（自信度标注偏乐观）、3.5（百分比制造精确性幻觉）

**根因：** `[90%]`/`[50%]`/`[猜]` 三档没有行为锚定，百分比暗示了不存在的精度。

**修补：**

1. **ops.md 步骤 6a 自信度标注从百分比改为行为锚：**
   ```
   [确定] = 能独立重构完整论证，不需要查任何资料
   [不确定] = 记得概念但无法完整重构论证链
   [推断] = 基于已有知识的合理推测，未经验证
   ```
   去掉数字百分比。

2. **QA 审查清单 #7（校准抽检）加打回判据：**
   ```
   从 PM 标了"确定"的论断中随机抽 2 处对照原文验证。
   自信度虚高 ≥2 处→标注"自评虚高"，打回重标
   ```

3. **研究路径常见坑加 DKE 提醒：**
   ```
   - 自信度假高 — 诚实标"不确定"强于虚标"确定"。QA 抽检发现虚标→打回。
     自信度标注的本质是 DKE（邓宁-克鲁格）防线——你不知道你不知道什么。
   ```

## P1 — PM 自审闸门

**覆盖瑕疵：** 4.3（PM 无自审闸门，步骤编号断档未自发现）、1.3（试点搜索未反馈到维度调整）、4.2（QA 存疑放行条件验收缺失）

**根因：** PM 拆完任务直接交 QA——QA 审查前 PM 自己不验证拆法质量。试点搜索的反馈也不回流到维度调整。

**修补：**

1. **ops.md 步骤 1 和步骤 2 之间加一步：**
   ```
   步骤 1.5：PM 自审拆法（不 spawn，PM 自己 3 分钟过一遍）
       对照 0.2 成功标准自问：
       - 步骤编号连续吗？
       - 每步有明确的输入输出吗？
       - 依赖关系标注了吗？
       - 失败路径有兜底方案吗？
       自审通过→交 QA。自审发现明显问题→先修再交
   ```

2. **ops.md 步骤 3 试点段补反馈回路：**
   ```
   试点返回后，PM 自问：试点结果是否暴露了维度划分的盲区？
   是否需要调整 A/B/C 路定义？
   如需调整→回步骤 1 重拆→步骤 2 QA 再审
   ```

3. **QA 存疑放行条件验收（补 ops.md 步骤 7 + QA agent）：**
   ```
   QA 存疑放行时条件必须明确化——"补 X 即过"。
   PM 补完后，QA 在下一轮审查中先验条件是否满足，再继续。
   条件验收不通过→仍算打回。
   ```
   QA agent 定义文件同步补：「存疑放行时，明确列出放行条件（可验证的具体条目）。下次 spawn 第一件事就是逐条验条件」

## P2 — 研究生水平操作化

**覆盖瑕疵：** 6.4（研究生水平从未操作化定义）、3.6（学习成果太干净）、3.7（无草稿→修订过程）、6.5（选题无价值判断）

**根因：** "研究生水平"四个字当万能标签用，没有可验证标准。学习成果格式不要求展示困惑和修订。

**修补：**

1. **ops.md 步骤 0.2 补操作化定义：**
   ```
   研究生水平 = 能通过该领域的博士资格考口试部分。
   操作化为 4 条可验证标准：
   ① 论证重构 — 能用自己的话独立重构核心论证链（从前提→结论）
   ② 批评理解 — 能对比至少 2 个主要批评者，指出其论证的强项和弱项
   ③ 跨学派对比 — 能对比至少 2 个相关学派/哲学家在该问题上的立场差异
   ④ 源追溯链 — 核心论断能追溯到具体的 L1/L2 一手来源
   每次研究任务 Step 0 把这 4 条写成针对该课题的具体自检标准
   ```

2. **ops.md 步骤 6c 补：**
   ```
   文档末尾必须有"学习困惑"节——记录学习过程中反复读但不完全理解的点、
   看起来自相矛盾但无法解决的张力。无困惑=学习不够深
   ```

3. **ops.md 步骤 6c 补修订痕迹：**
   ```
   PM 写完初稿后，至少间隔 10 分钟再自审一轮。
   自审发现的修正处标注 [修订]
   ```

4. **QA 审查清单补 #8：**
   ```
   #8 学习真实 — 是否包含"学习困惑"节？内容是真挣扎还是走过场？
   ```

5. **ops.md Step 0.2 补选题自问（不阻塞）：**
   ```
   可选自问：这个课题对 DK666 的实际工作有多少可迁移价值？
   PM 自答标注到任务笔记。DK666 看到后可叫停（正常档）。
   卡卡西模式下 PM 标注但不自行叫停
   ```

## P2 — 研究路径防漂移

**覆盖瑕疵：** 6.6（速度加快深度未跟）、3.9（34 分钟速度与深度矛盾）、6.7（任务笔记表演性）、5.4（"实验性 1/3"缺实验设计）

**根因：** 无趋势监控。口试"实验性"是计数器不是实验设计。笔记格式的完善程度可能掩盖任务实质质量。

**修补：**

1. **P1.15 测量框架加研究任务专项指标：**
   每次研究任务后记录：课题+耗时+源数量+口试结果（✅/⚠️/❌比例）+ PM 自评分数+QA 评分。
   月度看趋势。速度持续加快+QA 通过率不变→⚠️ 触发深度审查

2. **ops.md 研究路径步骤 7 补口试实验设计：**
   ```
   口试实验（1/3 窗口）：
   - 成功标准：口试能探到 PM 至少 1 处知识边界（不是 5/5 全对）
     AND QA 判定在后续留存测试中不被推翻
   - 失败标准：3 次口试全部 5/5 ✅ 且留存测试显示显著遗忘→题目难度偏低
   - 测量：每次口试后记录题目难度分布 + ✅/⚠️/❌ 比例 + PM 自评 vs QA 判定差异
   - 3 次实验后评估：保持/调整/放弃口试机制
   ```

3. **ops.md 常见坑补：**
   ```
   - 任务笔记完整度≠任务质量 — 格式完美的笔记+浅层执行 > 潦草笔记+扎实执行。
     PM 自问：笔记漂亮是因为任务做得好——还是模板设计得好？
   ```

4. **30d-system-full-review.md 加一行：**
   ```
   研究路径质量抽查 — 过去 30 天研究任务的口试通过率、PM 自评 vs QA 评分差异、
   单任务平均耗时趋势。口试通过率 100% + 速度持续加快 → ⚠️ 触发深度审查
   ```

## P2 — 合规+财务范围调整

   合规小克可豁免，但 PM 在步骤 5（拼合）时必须自检：
   54 源是否全部通过 AI 污染判据（有署名+有机构+有参考文献）？
   未通过的→标注"⚠️ 疑似AI生成"并排除。
   自检结果写入任务笔记

2. **ops.md 研究路径财务段改为：**
   ```
   财务小克事后 ROI（轻量版）：基于 spawn-cost.jsonl 自动采集的

   积累研究任务成本基线

## P3 — 已知限制声明

**覆盖瑕疵：** 1.5（三路独立性假象）、4.1（QA 无元审查）、5.3（人机结合悬置）、5.5（首次口试用卡卡西）、6.1（PM/QA 同模型）、6.2（单模型认知闭环）、6.3（研究路径是代码路径变体，改称"学习路径"待讨论）

**根因：** 这些都是架构天花板或本质性张力，不可在当前系统内消除。修补方式是透明化——让读者（DK666、未来会话的 AI）知道这些限制存在。

**修补：**

1. **protocol.md 加已知限制段（§九末尾）：**
   ```
   ### 已知架构限制
   - 所有角色使用同一底层模型。角色隔离靠 prompt，不等同于独立人类的独立判断。
