## 关键发现

### 1. macOS 26 内置听写对简体中文支持成熟
- **普通话（中国大陆）** 支持：设备端处理（Apple Silicon）、自动标点符号、听写表情符号、非模态听写（边听写边打字）
- **粤语（中国大陆/香港）** 同样支持设备端+自动标点
- **上海话** 仅支持云端听写，无设备端/自动标点
- Apple Silicon Mac 上中文听写请求在设备本地处理，不上传 Siri 服务器
- 自动标点支持中文的逗号、句号、问号自动插入

### 2. Universal Clipboard 是最简单的文字传输方式，但有约束
- 要求：**同一 Apple ID**、Wi-Fi+蓝牙均开启、Handoff 开启、两台设备在蓝牙范围内（~10米）
- **不要求同 Wi-Fi 网络**（设备通过蓝牙/Wi-Fi Direct P2P 通信）
- 剪贴板内容仅保留约 **2 分钟**，之后自动清除
- 数据端到端加密

### 3. 屏幕共享原生的"Use Shared Clipboard"可用于文字传输，但不传麦克风音频
- 内置 Screen Sharing.app 的 Edit > Use Shared Clipboard 支持 Mac-to-Mac **双向剪贴板同步**（文本+图片+文件拖放）
- 要求同局域网（或 VPN 连通），通过 `vnc://IP` 连接
- **关键限制**：屏幕共享**不传输客户端麦克风音频**到远程 Mac
- 即使 Apple Remote Desktop（$80）的高性能模式也只支持立体声音频**输出**，不含麦克风输入回传

### 4. AUNetSend/AUNetReceive 是 macOS 内置的网络音频传输方案，但维护差
- 通过 Bonjour 发现，Audio MIDI Setup 或 AU Lab 中可用
- 现代 macOS 上有 **IPv6 兼容问题**：AUNetReceive 一直显示 "connecting..."
- 变通方法：手动输入发送端 IPv4 地址:端口号，跳过 Bonjour
- Apple 多年未更新，社区长期呼吁开源

### 5. SonoBus 是目前最实用的免费跨 Mac 音频流方案
- 免费、开源、全平台（macOS/Windows/Linux/iOS/Android）
- 支持 **LAN 直连模式**（无需公网服务器）：输入对方 IP:端口即可
- P2P 架构，音频不经过第三方服务器
- 延迟：良好网络下 ~25-40ms 单向
- 要求：必须用有线耳机（避免回声），推荐有线网络
- 1.7.2 版本发布于 2023-12-11，功能稳定

## 方案汇总

### 方案 A：MacBook 本机听写 + Universal Clipboard 传文字

**流程：** MacBook 按两下 Fn 键触发听写 -> 口述中文 -> 文字出现在 MacBook 上 -> Cmd+C 复制 -> 在 Mac mini 上 Cmd+V 粘贴（通过 Universal Clipboard）

**中文识别质量：** 优（macOS 26 内置，设备端处理，自动标点）
**跨设备传输方式：** Universal Clipboard（同 Apple ID + Handoff + 蓝牙范围内）
**部署复杂度：** 低（两边都登录同一 Apple ID，开 Wi-Fi+蓝牙，开 Handoff，无需额外安装）
**是否需同局域网：** 否（蓝牙/Wi-Fi Direct P2P，但两台设备需在 ~10米内）
**已知限制：**
- MacBook 和 Mac mini 必须登录**同一个 Apple ID**（如果 Mac mini 用的是单独的 Apple ID 则不可行）
- 剪贴板只保留约 2 分钟，口述后需尽快粘贴
- Universal Clipboard 偶尔不稳定（部分用户报告间歇性失效）

### 方案 B：MacBook 本机听写 + 屏幕共享剪贴板传文字

**流程：** 打开 Screen Sharing.app 连接到 Mac mini -> MacBook 按 Fn 触发听写 -> 口述中文 -> 文字出现在 MacBook 编辑器中 -> 在 Screen Sharing 菜单 Edit > Use Shared Clipboard 开启共享剪贴板 -> Cmd+C -> 在远程 Mac mini 窗口 Cmd+V

**中文识别质量：** 优（同方案 A）
**跨设备传输方式：** Screen Sharing 内置"Use Shared Clipboard"（Mac-to-Mac VNC 原生功能）
**部署复杂度：** 低（Mac mini 开启屏幕共享，MacBook 用 Screen Sharing.app 连接，开启共享剪贴板）
**是否需同局域网：** 是（或 VPN 连通）
**已知限制：**
- 需要 Screen Sharing 会话保持打开
- 必须 Mac-to-Mac（第三方 VNC 客户端不行）
- 比 Universal Clipboard 多一步（需维持远程桌面连接）

### 方案 C：SonoBus 传音频 -> Mac mini 本机听写

**流程：** MacBook 装 SonoBus -> Mac mini 装 SonoBus -> 同局域网直连 -> MacBook 麦克风音频实时传到 Mac mini -> Mac mini 端打开任意文本编辑器 -> 触发 Mac mini 的听写功能 -> Mac mini 直接把收到的音频转成文字

**中文识别质量：** 优（Mac mini M4 Apple Silicon 设备端听写，同 macOS 26 引擎）
**跨设备传输方式：** SonoBus P2P 音频流（LAN 直连，无服务器中转）
**部署复杂度：** 中（两台 Mac 各装 SonoBus，配置直连 IP，Mac mini 端需耳机避免回声，需在 Mac mini 端手动触发听写）
**是否需同局域网：** 是
**已知限制：**
- MacBook 端必须用耳机（避免 MacBook 扬声器声音被 MacBook 自己的麦克风回收产生回声）
- 依赖第三方软件（非 macOS 原生）
- Mac mini 端听写只能持续 30 秒（超时自动停止），需反复触发
