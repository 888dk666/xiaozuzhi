## 二、重大发现（推翻原方案）

### 2.1 SubagentStart 不能 block spawn

查官方文档（code.claude.com/docs/en/hooks）后发现：SubagentStart **不支持 exit 2 拒绝 spawn**。官方明确——"Can block? No"，exit 2 仅显示 stderr，不阻止 spawn。后续通过 Jina（r.jina.ai）补抓获得完整文档，确认无遗漏。

**影响：** B 部分（活跃数硬挡）原方案不可行。降级为软提醒——stderr 警告 + additionalContext 提醒，硬挡靠 PM 自觉过自检清单。

### 2.2 SubagentStop 不支持 additionalContext

官方完整文档明确：SubagentStop 不支持 additionalContext。要向父会话注入内容需改用 PostToolUse hook（matcher: Agent 工具）。

### 2.3 晓组织没有 git repo

工程师 agent（engineer.md）配置了 `isolation: worktree`，但晓组织目录没有 `.git`——worktree 隔离从未实际生效。2026-05-25 初始化 git repo（4 commits），创建工程师 agent 软链接到 `~/.claude/agents/`。

## 十一、gap #8 实现可行性

### A. 成本日志 ✅ 完全可行

- SubagentStart hook：记时间戳 + agent_type + agent_id → 日志文件
- SubagentStop hook：记完成时间 + 读取 agent_transcript_path → 提取 token 消耗 + stop_reason
- 需设计：日志持久路径、日志格式、僵尸处理（SubagentStop 不触发时）
- 所有技术前提已验证

### B. 活跃数软提醒 ✅ 可行（降级版）

- SubagentStart hook：状态文件计数 + fcntl 锁 + ≥5 时 stderr 警告 + additionalContext 提醒
- SubagentStop hook：减计数 + fcntl 锁
- 需设计：状态文件路径、僵尸清理策略（agent 崩溃时计数漂移）
- 硬挡不可行（平台限制），软提醒可行
