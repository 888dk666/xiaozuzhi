## 关键发现

### 1. 讯飞输入法是 Android 中文语音识别的事实标准
官方宣称普通话准确率 98%，1 分钟 400 字，支持 23+ 种方言、离线识别（96%）、中英混合免切换。vivo 社区用户普遍反映"Jovi 输入法语音太烂，每次都要切到讯飞"。讯飞是 vivo 输入法设置中的可选语音引擎（"使用讯飞语音识别"开关），说明 vivo 自身也认可讯飞的技术优势。

### 2. vivo/Jovi 自带语音输入口碑很差
多个 vivo 社区用户反馈：识别率低、漏字严重（"今天你还好吗"变成"今天你好吗"）、标点不智能、响应慢。vivo 官方给出的优化建议之一是"切换为讯飞引擎" -- 等于默认承认不如讯飞。结论：不建议使用 vivo 自带语音输入。

### 3. KDE Connect 是 Android→Mac 文字传输的最佳免费方案
开源免费，同 WiFi 下 TLS 加密通信。Mac→手机剪贴板自动同步；手机→Mac 需手动点通知栏"发送剪贴板"（Android 安全限制）。但 macOS 26 (Sequoia) 上存在已知 Bug（阻止系统休眠），且有用户反馈稳定性问题。备选：ClipCascade（自部署，AES-256 加密，全平台客户端）、Snapdrop（浏览器打开即用，零安装）。

### 4. Android 手机当无线麦克风方案存在显著不确定性
- WO Mic：第三方来源称支持 macOS，但官方页面仅列 Windows Client。App Store 的 iOS 版标注 macOS 11.0+ 兼容（即 iOS App 可在 Apple Silicon Mac 运行），但这是否适用于 Android→Mac 路径存疑。此矛盾未解决，标注待验证。
- DroidCam：确认无 macOS 独立客户端，仅支持 OBS 插件，需安装 OBS Studio 再用虚拟摄像头桥接，部署复杂度对电脑小白太高。

### 5. scrcpy 提供最强的技术方案但部署门槛高
scrcpy 通过 ADB 连接，可实现 Android↔电脑剪贴板全自动双向同步、文字注入（键盘序列模拟），功能最强。但需要手机开启开发者选项+USB 调试，Mac 端通过 `brew install scrcpy` 安装。适合有技术背景的用户，对电脑小白不友好。

## 方案汇总

### 方案 B-1：讯飞输入法语音转文字 + KDE Connect 传输
- **中文识别质量：** 优 -- 讯飞输入法，普通话 98%，23+ 方言，离线可用
- **跨设备传输方式：** 手机讯飞语音→文字出现在输入框→全选复制→KDE Connect 通知栏点"发送剪贴板"→Mac mini 端 Cmd+V 粘贴
- **延迟：** 低 -- 语音识别实时出字 + 剪贴板手动发送约 1-2 秒
- **部署复杂度：** 中 -- 手机装讯飞+KDE Connect，Mac 装 KDE Connect 或 Soduto，同 WiFi 配对
- **是否需同局域网：** 是（KDE Connect 要求同 WiFi）
- **是否免费：** 是 -- 讯飞免费，KDE Connect 开源免费
- **已知风险：** macOS 26 上 KDE Connect 有休眠 Bug（TechPP 2025/10 报道）；手机→Mac 需手动触发（非自动化）
- **简化变体：** 不用 KDE Connect，手机语音→文字后直接发微信/Telegram"文件传输助手"→Mac 端同步粘贴。部署更简单（零额外安装），但多一步"打开微信发送"

### 方案 B-2：Gboard 语音输入 + Snapdrop 传输
- **中文识别质量：** 优 -- Gboard 安卓端语音识别公认最强之一，粤语实测超 90%，支持 900+ 语言变体
- **跨设备传输方式：** 手机 Gboard 语音→文字→复制→Snapdrop（浏览器打开 snapdrop.net）→点击 Mac 图标→粘贴→Mac 端浏览器接收→复制
- **延迟：** 中 -- 语音识别实时 + Snapdrop P2P 传输约 2-3 秒
- **部署复杂度：** 低 -- 手机装 Gboard（Google Play 直接下载），两端浏览器打开 snapdrop.net 即可
- **是否需同局域网：** 是（Snapdrop 靠 WebRTC 局域网 P2P）
- **是否免费：** 是 -- Gboard 免费，Snapdrop 开源免费
- **优势：** 零安装（双方都是浏览器），比 KDE Connect 更稳定（无 macOS 兼容问题）
- **劣势：** 每次传文字要打开浏览器；Gboard 离线语音仅 Pixel 支持，vivo 需联网

### 方案 B-3：讯飞/Gboard 语音 + 微信"文件传输助手"
- **中文识别质量：** 优 -- 同 B-1 或 B-2 任选
- **跨设备传输方式：** 手机讯飞/Gboard 语音→文字→全选复制→打开微信→发给"文件传输助手"→Mac 微信同步收到→复制粘贴到目标位置
- **延迟：** 低 -- 语音识别实时 + 微信消息同步约 1-2 秒
- **部署复杂度：** 低 -- 手机和 Mac 都已有微信，不需额外安装任何工具
- **是否需同局域网：** 否 -- 微信通过互联网同步，随处可用
- **是否免费：** 是 -- 微信免费
- **优势：** 最简单，没有任何部署成本，跨网络；微信是 DK666 已在用的工具
- **劣势：** 依赖微信（隐私考虑），文字混在聊天记录里不够"干净"；Mac mini 远程操作时需确认微信已登录
